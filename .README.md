## Overview

FinanceHub is a full-stack web application designed to process, visualize, and control access to financial data within an organization. It implements a three-tier role model (Viewer → Analyst → Admin) enforced on both the frontend (protected routes) and the backend (middleware-level authorization), ensuring users can only access and manipulate data appropriate to their role.

---

## ✨ Features

### 🔐 Authentication & Security
- User registration & login with **bcrypt**-hashed passwords
- **JWT** (JSON Web Token) issuance on login, sent as a `Bearer` token
- Token auto-attached to every API request via **Axios interceptors**
- Auto-logout on `401 Unauthorized` response — token & user data cleared from `localStorage`
- Inactive account blocking at the middleware level

### 🛡️ Role-Based Access Control (RBAC)
- Three roles: `viewer`, `analyst`, `admin`
- **Frontend**: React `<ProtectedRoute>` component blocks unauthorized page access
- **Backend**: `roleMiddleware` enforces allowed roles per route before the controller executes

### 📊 Dashboard Analytics
- Total Income, Total Expense, Net Balance stat cards
- **Category breakdown**: spending/income aggregated per category
- **Monthly trends**: configurable time-window (up to 24 months) with income vs. expense comparison
- All analytics are company-wide aggregations powered by MongoDB's aggregation pipeline

### 📁 Financial Records (CRUD)
- Create, read, update, and delete financial records (admin only for mutations)
- Fields: amount, type (income/expense), category, date, optional note
- Filter & search records by type, category, and date range
- Paginated listing with real-time feedback via toast notifications

### 🎨 Premium UI/UX
- Dark-mode first design with glassmorphism cards and gradient accents
- Smooth animations, hover micro-interactions, and loading states
- Responsive layout across all screen sizes
- One-click **demo account chips** on the login page for instant access

## 🏗️ System Architecture

```
┌─────────────────────────────────────────────┐
│              React Frontend (Vite)          │
│                                             │
│  Login / Register   →   AuthContext (JWT)   │
│         ↓                                   │
│  ProtectedRoute (role check)                │
│    ├── /dashboard  →  Dashboard.jsx         │
│    └── /records    →  Records.jsx           │
│                                             │
│  Axios Instance  →  Bearer Token Header     │
└──────────────────────┬──────────────────────┘
                       │ HTTPS / REST
┌──────────────────────▼──────────────────────┐
│           Express API Server                │
│                                             │
│  POST /api/auth/register                    │
│  POST /api/auth/login                       │
│                                             │
│  authMiddleware (JWT verify)                │
│       ↓                                     │
│  roleMiddleware (role check)                │
│       ↓                                     │
│  Controller → Mongoose → MongoDB Atlas      │
│                                             │
│  /api/records     (analyst, admin)          │
│  /api/dashboard/* (viewer, analyst, admin)  │
└─────────────────────────────────────────────┘
```

## 🛡️ Role-Based Access Control

| Endpoint / Feature | Viewer | Analyst | Admin |
|---|:---:|:---:|:---:|
| Login / Register | ✅ | ✅ | ✅ |
| View Dashboard Stats | ✅ | ✅ | ✅ |
| View Monthly Trends | ✅ | ✅ | ✅ |
| View Category Stats | ✅ | ✅ | ✅ |
| View Records List | ❌ | ✅ | ✅ |
| View Single Record | ❌ | ✅ | ✅ |
| **Create** Record | ❌ | ❌ | ✅ |
| **Update** Record | ❌ | ❌ | ✅ |
| **Delete** Record | ❌ | ❌ | ✅ |

Authorization is enforced at **two layers**:
1. **Frontend** — `ProtectedRoute` redirects unauthorized users before the page even loads.
2. **Backend** — `roleMiddleware` returns `403 Forbidden` if the JWT role doesn't match the route's allowed roles, ensuring the API is secure even if the frontend is bypassed.

## 📡 API Reference

### Auth

| Method | Endpoint | Body | Description |
|---|---|---|---|
| `POST` | `/api/auth/register` | `{ name, email, password, role? }` | Register a new user |
| `POST` | `/api/auth/login` | `{ email, password }` | Login, returns JWT token |

### Dashboard *(All authenticated roles)*

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/dashboard/income` | Total income across all records |
| `GET` | `/api/dashboard/expense` | Total expense across all records |
| `GET` | `/api/dashboard/balance` | Net balance (income − expense) |
| `GET` | `/api/dashboard/category` | Spending/income grouped by category |
| `GET` | `/api/dashboard/trends?months=12` | Monthly income & expense trends |

### Records *(Analyst & Admin)*

| Method | Endpoint | Roles | Description |
|---|---|---|---|
| `GET` | `/api/records` | analyst, admin | List all financial records |
| `POST` | `/api/records` | admin | Create a new record |
| `GET` | `/api/records/:id` | analyst, admin | Get a single record by ID |
| `PATCH` | `/api/records/:id` | admin | Update a record |
| `DELETE` | `/api/records/:id` | admin | Delete a record |

All protected routes require the header:
```
Authorization: Bearer <jwt_token>
```

## 🗃️ Data Models

### User

```js
{
  name:      String,   // required
  email:     String,   // required, unique, lowercase
  password:  String,   // bcrypt hashed, stripped from all JSON responses
  role:      "viewer" | "analyst" | "admin",   // default: "viewer"
  status:    "active" | "inactive",            // default: "active"
  createdAt: Date,
  updatedAt: Date
}
```

### Record (Financial Transaction)

```js
{
  userId:    ObjectId,         // ref → User
  amount:    Number,           // required
  type:      "income" | "expense",  // required
  category:  String,           // required (e.g. "Salary", "Rent", "Food")
  date:      Date,             // default: Date.now
  note:      String,           // optional
  createdAt: Date,
  updatedAt: Date
}
```

## 🚀 Getting Started

### Prerequisites
- **Node.js** v18+
- **npm** v9+
- A **MongoDB Atlas** cluster (or local MongoDB)

### 1. Clone the Repository

```bash
git clone <repo-url>
cd "Finance Data Processing and Access Control Backend"
```

### 2. Setup & Run the Backend

```bash
cd server
npm install
```

Create a `.env` file in the `server/` directory (see [Environment Variables](#environment-variables) below), then:

```bash
# Development (with hot-reload)
npm run dev

# Production
npm start
```

The API will start on `http://localhost:5000`.

### 3. Setup & Run the Frontend

```bash
cd cilent/finance-frontend
npm install
npm run dev
```

The frontend will start on `http://localhost:5173`.

## 🔧 Environment Variables

Create a `.env` file in the `server/` directory:

```env
PORT=5000
MONGO_URI=mongodb+srv://<username>:<password>@<cluster>.mongodb.net/<dbname>
JWT_SECRET=your_super_secret_jwt_key
```

| Variable | Description |
|---|---|
| `PORT` | Port the Express server listens on (default: `5000`) |
| `MONGO_URI` | MongoDB connection string (Atlas or local) |
| `JWT_SECRET` | Secret key used to sign & verify JWT tokens |
